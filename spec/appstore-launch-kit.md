## 6. Launch-day runbook (Phase 2 W5; target Tue Aug 18 / Wed Aug 19)

**T-7 → T-1 (W4):** build submitted with §5 notes · press kit final (founder
photo taken) · r/golf post final + mods messaged · IG/TikTok/X launch posts
drafted · newsletter intro emails sent (N4/N5 warm-ups) · flyer file at the
printer · white-glove calendar cleared for W5–W6.

**Launch day (in order):**
1. Confirm the app is live in the store; install on a clean phone; create a
   league end-to-end; check the version caption + console.
2. Release the phased-rollout hold if used; screenshot the listing.
3. PIGL board post (the crew hears first — always).
4. r/golf founder post (morning MST) — then live in the comments all day.
5. IG/TikTok launch story + settlement-card post; X mirror.
6. Personal texts to every warm Pro and Founding candidate with the store
   link ("it's live; want me to set your season up this week?").
7. Press kit + store link to the newsletter/podcast list (N1–N7 per
   outreach doc timing).
8. Evening: first-day numbers into the Monday-report format; triage list.

**Triage rules:** review rejection on money grounds → respond with §5
language, escalate to a call with App Review, do NOT soften the app's
ledger surfaces without a decision-log entry (that's a mechanic/voice canon
change, not a support ticket). Crash/bug surge → the eng lane owns the fix;
socials go quiet rather than posting over a broken door. Store approved but
web lagging → remember the two-deploy discipline; the PWA and store build
are separate ships.

**What we do NOT do at launch:** Product Hunt (wrong audience for founder
hours) · paid anything · pricing talk in any reply · follower-count posts.

## 7. Eng-lane dependencies (flagged, not built here)

   native push, share-sheet integration, and a real icon set so the app
   feels native, not a browser tab. (Push infra already exists server-side.)

   league.

   details, user-content photos; no tracking, no third-party ads. Complete
   in App Store Connect at submission; keep consistent with legal.html.
4. **Version discipline** — `__CS_VERSION__` stamping stays build-owned in
   the wrapper exactly as on web (CLAUDE.md rule 2).
5. **Artifact→claim instrumentation** (separate task, already filed) — the
   launch week's share numbers are dark without it; it is the measurement
   plan's first dependency, not a launch blocker.
